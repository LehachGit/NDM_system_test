Что это
Стенд демонстрирует корректную передачу заголовка X-Forwarded-For через цепочку
nginx reverse proxy с защитой от подделки со стороны клиента.
Развёрнуто 3 контейнера nginx (минимум по заданию) и балансировщик,
который распределяет входящие запросы между ними по алгоритму round-robin —
то есть каждый новый запрос попадает на следующий nginx по кругу.
Приложение на каждый запрос показывает:

Client IP — реальный IP того, кто обратился
Proxy chain — полный путь запроса: через какие nginx он прошёл до приложения

Запуск
bashdocker-compose up -d
Приложение будет доступно по адресу http://<ip>:8080.
Если порт занят — поменяй 8080 на любой свободный в docker-compose.yml.
Проверка
Открой http://<ip>:8080 в браузере или через curl:
bashcurl http://localhost:8080/
В ответе увидишь свой IP и цепочку прокси через которые прошёл запрос.
Проверка защиты от подделки — приложение должно проигнорировать поддельный заголовок:
bash# Попытка подделать чужой IP
curl http://localhost:8080/ -H "X-Forwarded-For: 6.6.6.6"

# Попытка подделать IP одного из наших nginx
curl http://localhost:8080/ -H "X-Forwarded-For: 172.20.0.11"
В обоих случаях поддельный IP не появится в выводе.
